# Автоматизация томов НДВ / НДС / СЗЗ в ЭкоДок

Тома НДВ/НДС/СЗЗ — «частично автоматизируемые»: расчёт рассеивания и физфакторов
делает специализированное ПО (УПРЗА «Эколог», «Эколог-Шум», «Эколог-ПДВ» и пр.),
его ЭкоДок **не заменяет**. Но это не значит, что автоматизировать нечего —
наоборот, большая часть трудозатрат тома лежит вне расчёта.

## 1. Где в томе ручной труд (и что забирает автоматизация)

| Часть тома | Источник | Кто делает | Доля объёма |
|---|---|---|---|
| Титул, аннотация, реквизиты предприятия | ReportContext | **ЭкоДок (авто)** | ~10% |
| Сведения о предприятии, площадке, режиме | ReportContext + анкета | **ЭкоДок (авто)** | ~10% |
| Таблица параметров источников выбросов | инвентаризация / «Эколог» | импорт из «Эколога» | ~15% |
| Расчёт рассеивания, макс. концентрации | **«Эколог»/УПРЗА** | внешнее ПО | ядро (нельзя авто) |
| Карты изолиний, ситуационный план | «Эколог» / ГИС | внешнее ПО | приложение |
| Обоснование нормативов, текст разделов | шаблон + результаты | **ЭкоДок (авто)** | ~25% |
| Мероприятия, выводы | шаблон + условия | **ЭкоДок (полуавто)** | ~10% |
| Протоколы КХА/биотестирования | лаборатория | подгрузка в приложения | приложение |

Вывод: **расчётное ядро — у «Эколога», но ~55–65% тома (текст, реквизиты,
сборка таблиц, приложения) автоматизируется.** Это и есть зона ЭкоДок.

## 2. Ключевая идея — «инвентаризация один раз»

Главная боль — двойной/тройной ввод одних и тех же данных об источниках в
ЭкоДок, в «Эколог» и в текст тома. Решение — сделать ЭкоДок **единым местом
ввода инвентаризации**, а «Эколог» — расчётным «придатком»:

```
   ┌─────────────┐   экспорт источников   ┌──────────────┐
   │   ЭкоДок     │ ─────────────────────▶ │  «Эколог»    │  расчёт рассеивания
   │ инвентари-   │                        │  (УПРЗА)     │  (нельзя авто)
   │ зация (SoT)  │ ◀───────────────────── │              │
   └─────────────┘   импорт результатов    └──────────────┘
         │ + реквизиты, шаблоны, приложения (КХА)
         ▼
   готовый том НДВ/НДС/СЗЗ (.docx)  — собирается автоматически
```

- **Экспорт в «Эколог»**: ЭкоДок отдаёт таблицу источников в формате, который
  «Эколог» импортирует (Excel; в перспективе — нативный обменный формат «Эколога»).
- **Импорт из «Эколог»**: «Эколог» выгружает таблицы источников и результаты
  рассеивания в Excel/RTF → `volume_builder.ingest_excel()` затягивает их в том.
- **Приложения**: протоколы КХА, биотестирование, карты — подгружаются как
  исходники (`parsers.text_extract` читает pdf/jpg/docx), попадают в раздел
  «Приложения» по списку.

## 3. Что уже реализовано (каркас)

`ecodoc/development/volume_builder.py`:
- `build("ndv"|"nds"|"szz", ctx, sources, out)` — собирает .docx со всеми
  типовыми разделами, титулом и реквизитами из `ReportContext`;
- `ingest_excel(path)` — импорт таблицы источников / результатов рассеивания из
  выгрузки «Эколога»;
- `collect_appendices(paths)` — реестр приложений-исходников;
- CLI: `python -m ecodoc volume ndv -i context.json --sources-xlsx ист.xlsx
  --dispersion-xlsx рассеивание.xlsx --appendix КХА1.pdf КХА2.pdf`.

Сейчас при отсутствии выгрузок «Эколога» в томе стоят явные плейсхолдеры
(`‹импортировать из «Эколога»›`) — структура готова, наполнение подставляется.

## 4. Дорожная карта тома (по приоритету)

1. **Модель инвентаризации источников** в `ReportContext` (источники выброса:
   №, координаты,高 H, диаметр D, скорость/температура ГВС, перечень веществ,
   г/с и т/год) — чтобы вводить один раз. Сейчас источники приходят таблицей
   из «Эколога»; цель — наоборот.
2. **Экспорт инвентаризации в импортируемый «Экологом» формат** (Excel-шаблон,
   далее — нативный обмен, если документирован).
3. **Парсеры выгрузок «Эколога»** под конкретные его экспортные формы (шапки
   таблиц источников и результатов) — маппинг колонок.
4. **Шаблоны разделов** под НДВ / НДС / СЗЗ раздельно (тексты-обоснования,
   ссылки на НПА, условные блоки «при превышении → мероприятия»).
5. **Сборка приложений**: вшивание протоколов КХА (или ссылок), карт изолиний
   (вставка изображений из «Эколога»).
6. **СЗЗ-специфика**: объединение рассеивания («Эколог») + акустики
   («Эколог-Шум») + ЭМИ → единое обоснование размера СЗЗ + заявление в РПН/РПН.

## 5. Чего автоматизация не сделает (честно)

- Сам расчёт рассеивания/шума — только «Эколог»/УПРЗА (методики, аттестованное ПО).
- Определение класса опасности отхода — лаборатория (КХА) + расчёт.
- Согласования и экспертные решения по мероприятиям.

ЭкоДок убирает рутину (ввод, таблицы, текст, сборка, приложения), оставляя
эксперту расчёт и решения.
