# ЭкоДок — анализ: документы эксплуатации, схема создания, подача в ЛКПП, масштаб на много организаций

Дата: 30.06.2026. Источник правды по срокам/форматам — действующие НПА и
`lk.rpn.gov.ru/instructions` + `/operators-page` (проверять ежегодно).

---

## 1. Полный реестр обязательных экологических документов (эксплуатация)

Документы делятся на **постоянные/разрешительные** (делаются один раз и
актуализируются по событию) и **периодическую отчётность** (сдаётся по графику).
Состав зависит от **категории объекта НВОС** (I / II / III / IV).

### 1.1. Постоянные / разрешительные (основа, из них берутся данные для отчётности)

| Документ | Кому нужен | Как делается | Автоматизация в ЭкоДок |
|---|---|---|---|
| **Постановка на учёт объекта НВОС** (заявка), актуализация сведений | все, кто оказывает НВОС | заявка через ЛКПП, присваивается код объекта и категория | средняя: сбор реквизитов + XML заявки |
| **Паспорта отходов I–IV класса** | все ЮЛ/ИП, у кого образуются отходы I–IV кл. | по ФККО (типовая форма) или по результатам определения класса (лаб. + расчёт) | **высокая**: для отходов из ФККО — генерация по шаблону |
| **Сведения о компонентном составе отхода** (для отходов не из ФККО) | при отходах, не внесённых в ФККО | протокол КХА + расчёт класса опасности | низкая (нужна лаборатория) |
| **Проект НДВ** (нормативы допустимых выбросов) | I, II категории | расчёт рассеивания в ПО «Эколог»/УПРЗА | низкая (внешнее ПО), ЭкоДок — оформление тома |
| **Проект НДС** (нормативы допустимых сбросов) | при сбросе в водные объекты | расчёт, согласование | низкая |
| **Проект СЗЗ** (санитарно-защитная зона) | при наличии воздействия (шум, выбросы) | расчёт рассеивания + физфакторы, согласование с РПН/Роспотребнадзором | низкая–средняя (оформление) |
| **НООЛР** (нормативы образования отходов и лимиты) | I, II категории | расчёт нормативов | средняя (расчёт по справочникам) |
| **Программа ПЭК** | I, II, III категории | по Приказу №109: разделы, точки контроля, графики | **высокая**: генерация по шаблону из перечня источников/отходов |
| **КЭР** (комплексное экологическое разрешение) | I категория | большой пакет, заявка через ЛКПП | низкая (комплексный проект) |
| **ДВОС** (декларация о воздействии) | II категория | заявка через ЛКПП | средняя |

### 1.2. Периодическая отчётность (целевой контур «Отчётность» ЭкоДок)

| Отчёт | Кому | Периодичность / срок | Куда | Автоматизация |
|---|---|---|---|---|
| **Декларация о плате за НВОС** | I, II, III (IV — **не платят**) | год, **до 10 марта** | ЛКПП (РПН) | **полная** ✓ реализовано |
| **Авансовые платежи за НВОС** | те же, кроме МСП | квартал, до 20 числа след. месяца | расчёт платежа | **полная** (тот же движок) |
| **Отчёт ПЭК** | I, II, III | год, **до 25 марта** | ЛКПП (РПН) | высокая (каркас есть) |
| **2-ТП (отходы)** | при образовании/обращении | год, **до 1 февраля** | ЛКПП, **есть API** | высокая ✓ модель есть |
| **2-ТП (воздух)** | при стац. источниках | год, **до 22 января** | ЛКПП, **есть API** | высокая |
| **2-ТП (водхоз)** | при водопользовании | год, **до 22 января** | Росводресурсы/ЛКПП | средняя |
| **Учёт в области обращения с отходами (движение)** | все, у кого отходы | ведётся постоянно, данные — поквартально | хранится у предприятия (предъявляется по запросу) | высокая ✓ реализовано |
| **Отчётность об образовании, утилизации… отходов (III кат./МСП)** | III категория и субъекты МСП (вместо НООЛР) | год, до 25 января | ЛКПП (РПН) | высокая |
| **Региональный кадастр отходов** | по региону (СПб/ЛО) | год, региональный срок | региональный портал | средняя (каркас есть) |
| **4-ОС, 2-ОС, 4-ЛС, 2-Медотходы и пр.** | по виду деятельности | разные | Росстат/РПН | по необходимости |

**Категорийная логика (ядро автоопределения состава):**
- **IV категория** — самый лёгкий режим: только паспорта отходов + учёт движения;
  декларация и плата НВОС **не подаются**, ПЭК не требуется, 2-ТП — по факту отходов.
- **III категория** — декларация НВОС, ПЭК, 2-ТП, отчётность по отходам (вместо НООЛР).
- **I, II категории** — всё вышеперечисленное + проекты (НДВ/НДС/НООЛР), КЭР (I)/ДВОС (II).

## 2. Как делать документы и что реально автоматизируется

Три класса по степени автоматизации:

1. **Полностью автоматизируемо (правила + справочники + данные):**
   декларация и плата НВОС, учёт движения отходов, 2-ТП (отходы/воздух),
   паспорта отходов из ФККО, программа и отчёт ПЭК (по шаблону), отчётность
   III категории по отходам, кадастр. → **это и есть ядро ЭкоДок.**

2. **Частично (нужен расчёт во внешнем ПО):** проекты НДВ/НДС/СЗЗ требуют
   рассеивания (ПО «Эколог»/УПРЗА). ЭкоДок здесь — оформление тома, проверка
   комплектности, перенос результатов, но не сам расчёт рассеивания.

3. **Нельзя без человека/лаборатории:** определение класса опасности отхода
   (КХА), согласования. ЭкоДок — подготовка заявок и сопроводительных.

**Единый принцип:** один раз введённые/распознанные данные об объекте, источниках,
отходах (= `ReportContext`) используются всеми формами. Декларация, ПЭК, 2-ТП и
учёт движения берут отходы и выбросы из одной модели — двойного ввода нет.

## 3. Нужна ли PDF-версия

Разделяем три выхода и не путаем их назначение:

| Выход | Назначение | Когда обязателен |
|---|---|---|
| **XML** | юридически значимая подача в ЛКПП/по API | всегда для подачи в РПН |
| **Excel/Word (печать)** | проверка человеком, согласование с клиентом, рабочая копия | для контроля и сдачи на бумаге (редко) |
| **PDF** | архив, отправка клиенту «как есть», бумажная подпись, приложения | **по необходимости**, не для подачи |

Вывод: **PDF не нужен для подачи** (РПН принимает XML; печатная форма для глаз —
Excel/Word). PDF полезен как:
- финальный «снимок» декларации/отчёта с подписью и печатью для архива и клиента;
- сшивка приложений (паспорта, протоколы) в один файл.

→ В ЭкоДок добавить PDF как **опциональный экспорт** печатной формы
(Excel/Word → PDF), а не как обязательный шаг. Для пакетной рассылки клиентам —
да; для подачи в кабинет — нет.

## 4. Загрузка отчётности в кабинет и автоматизация подачи

### 4.1. Способы подачи в ЛКПП (lk.rpn.gov.ru)

1. **Ручной ввод** в веб-кабинете — медленно, не масштабируется.
2. **Загрузка готового XML** через веб-интерфейс — ЭкоДок генерирует XML,
   человек заходит и грузит файл. Работает уже сейчас для всех форм.
3. **API (пакетный приём)** — официальный, машинный. Сейчас реализован для
   **2-ТП (воздух)** и **2-ТП (отходы)**; перечень форм расширяется.
4. **«Модуль природопользователя»** (десктоп) — **заморожен, не обновляется**,
   ставку на него не делаем.

### 4.2. Доступ к API

Заявка в РПН на `OD@rpn.gov.ru` или через `rpn.gov.ru/petition` с указанием
**ОГРН, ФИО и e-mail аккаунта** ЛК, для которого настраивается интеграция.
Формат XML — `lk.rpn.gov.ru/operators-page`. Подача возможна **с ЭЦП и без**;
юридически значимо — с квалифицированной ЭЦП или по **МЧД**.

### 4.3. Юридический ключ к масштабу — ЭЦП и МЧД

Главное ограничение автоматизации — **подпись**. Нельзя «程序но» подписать за
клиента без его ключа. Два легальных пути для консультанта, ведущего много
организаций:

- **МЧД (машиночитаемая доверенность)** — клиент выдаёт доверенность на
  представителя (вас/сотрудника) с нужными кодами полномочий для РПН; вы
  подписываете отчётность **своей** ЭЦП от имени многих клиентов. Это штатный
  механизм 2026 года и основа multi-org сценария.
- **ЭЦП клиента** — если клиент передал токен/облачную подпись (реже, рискованно).

### 4.4. Как автоматизировать при многих организациях

```
[Реестр организаций]  → для каждой: профиль (реквизиты, категория, объекты,
                         ОКТМО, доступ к ЛК, МЧД, какие формы и сроки)
        │
        ▼
[Сбор исходных данных] → парсинг приложенных doc/pdf/jpg + выгрузки клиента
        │                 (объёмы отходов, выбросы) → ReportContext на каждую орг.
        ▼
[Генерация пакета]    → по календарю: для каждой орг. нужные XML + печать (+PDF)
        │                 валидация по XSD до отправки
        ▼
[Подача]              → 3 уровня:
        ├─ API (где доступно): автоматическая пакетная отправка по токену
        ├─ XML-загрузка: подготовленные файлы + чек-лист «зайти и загрузить»
        └─ подпись: своей ЭЦП по МЧД (полуавтомат — клик подписи)
        ▼
[Контроль статусов]   → журнал: сгенерировано / подписано / отправлено /
                         принято / квитанция; напоминания о сроках
```

**Практический поэтапный план автоматизации подачи:**
- **Этап 1 (сейчас):** ЭкоДок генерирует валидный XML + печать для всех форм →
  человек грузит в ЛК. Уже даёт основной выигрыш по времени.
- **Этап 2:** получить доступ к API РПН (заявка), автоматизировать подачу 2-ТП
  (отходы/воздух) пакетно по реестру организаций.
- **Этап 3:** по мере расширения API РПН — подключать остальные формы; для
  не-API форм оставить полуавтомат (RPA-помощник + МЧД-подпись).
- **RPA-резерв:** там, где API нет, — браузерная автоматизация ЛК (есть наработки
  claude-browser-mcp / Playwright) строго как помощник под контролем человека на
  шаге подписи.

## 5. Общая схема работы пакета ЭкоДок (оценка и целевая архитектура)

### 5.1. Слои

```
                ┌─────────────────────────────────────────────┐
                │  CLI / (позже) веб-UI / пакетный планировщик  │
                └─────────────────────────────────────────────┘
                                   │
   ┌───────────────┬──────────────┼───────────────┬────────────────┐
   ▼               ▼              ▼               ▼                ▼
[Парсинг]    [ReportContext]   [Справочники]   [Формы]        [Подача]
doc/pdf/jpg   единая модель     ставки/ФККО/    declaration/    XML→ЛКПП:
+OCR →        организации,      коэффициенты,    pek/2-tp/       API | загрузка
извлечение    объектов,         категории        учёт/кадастр    | МЧД-подпись
реквизитов    отходов,          (data/*.json)   (XML+печать+PDF) + статусы
              выбросов
   │               │                                │
   └──── провенанс (откуда значение) ───────────────┘
                                   │
                          [Валидация: бизнес-правила + XSD РПН]
```

### 5.2. Что уже есть (v0.1.0) и насколько ложится в схему

- ✓ Единая модель `ReportContext`, реестр форм, деньги на `Decimal`,
  справочники в JSON, JSON-сериализация контекста (= «форма ручной доводки»).
- ✓ Парсинг doc/pdf/jpg + OCR, эвристика реквизитов с провенансом.
- ✓ Декларация НВОС — полностью (расчёт + XML + Excel). Учёт движения — XML+Excel+баланс.
- ✓ Рендер XML и Excel, заготовка валидации по XSD.
- ○ Каркасы: ПЭК, кадастр СПб.

Архитектура **уже соответствует** целевой схеме на уровне одной организации.
Чего не хватает для полной картины из этого анализа:

### 5.3. Дорожная карта (приоритеты)

1. **Автоопределение состава отчётности по категории** (I/II/III/IV) — модуль
   «что вы обязаны сдать и когда» из профиля организации. Высокая ценность,
   низкая трудоёмкость.
2. **Многоорганизационный слой:** реестр организаций (`orgs/*.json` или БД),
   пакетный прогон, календарь сроков, журнал статусов подачи.
3. **Реальные XSD РПН** для каждой формы → жёсткая валидация перед отправкой
   (формат XML — `lk.rpn.gov.ru/operators-page`).
4. **Дореализация форм:** ПЭК (Приказ №109), 2-ТП (отходы/воздух), отчётность
   III категории, кадастр СПб.
5. **Интеграция с API РПН** (после получения доступа) — авто-подача 2-ТП.
6. **PDF-экспорт** печатных форм (опционально) + сшивка приложений.
7. **Паспорта отходов из ФККО** — генератор по шаблону (мост к разрешительному
   контуру, переиспользует перечень отходов из `ReportContext`).
8. **МЧД-помощник:** хранение доверенностей по организациям, подстановка
   полномочий, полуавтомат подписи.

### 5.4. Риски и ограничения (честно)

- **Подпись (ЭЦП/МЧД)** — принципиально не полностью автоматизируема; всегда
  останется шаг с участием человека/ключа. Это правовое, не техническое
  ограничение.
- **Ставки/коэффициенты/сроки меняются ежегодно** — держать в `data/*.json`
  и сверять перед сезоном (январь–март).
- **API РПН покрывает не все формы** — гибрид «API + XML-загрузка + RPA» ещё
  надолго.
- **Региональные формы (кадастр)** — отдельные форматы и порталы по субъектам.
