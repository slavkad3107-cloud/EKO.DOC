# ЭКО.DOC — продукт «ЭКО Проект»

**Версия: 0.10.0** · [github.com/slavkad3107-cloud/EKO.DOC](https://github.com/slavkad3107-cloud/EKO.DOC)
· zip-архив каждой версии собирается `release.ps1` (номер версии — в имени файла)

Генератор экологических документов с подключаемыми модулями. Каждая форма
выдаёт **XML** для загрузки в ЛКПП / Модуль природопользователя
Росприроднадзора, **печатную форму** (Excel/Word) и по желанию **PDF**.

Контуры-модули: **«Отчётность»**, **«Разработка»**, **«Календарь подачи»**,
**«Приём документов»** (intake), **«ИИ-анализ»** (ai), **«Сторож форм»**
(watch), **рабочее пространство** организации × площадки. Загрузка в ЛКПП —
отдельный поздний контур; сейчас фокус на генерации документов.

## Графический интерфейс

```bash
ecodoc gui
```

Откроется браузер (127.0.0.1:8737): организации и площадки слева; вкладки —
Приём документов (загрузка + ИИ-анализ с цитатами), Данные (правка
context.json), Генерация (все формы + PDF в один клик), Календарь,
Рассеивание, Сервис (настройка ИИ, сторож форм). Данные площадок хранятся
в `%USERPROFILE%\ЭКО.DOC`.

## Быстрый старт в консоли (мульти-организации + ИИ)

```bash
# Один раз: найти локальные ИИ (Ollama/LM Studio) и ключи внешних API
python -m ecodoc ai setup

# Завести организацию и площадку
python -m ecodoc org add "ООО Ромашка" --inn 7801234567 --oktmo 40303000
python -m ecodoc site add "ООО Ромашка" "Площадка №1"

# Принести документы: что распознано и откуда взято, чего не хватает
python -m ecodoc intake "справка_утилизация.pdf" "протокол_КХА.pdf" \
    --org "ООО Ромашка" --site "Площадка №1" --ai

# Сгенерировать форму (XML + Excel + PDF) в out/ площадки
python -m ecodoc generate declaration-nvos --org "ООО Ромашка" --site "Площадка №1" --pdf

# Перерасчёт при новых данных: просто intake новых документов + generate заново

# Проверить, не изменились ли формы/ставки/схемы загрузки
python -m ecodoc watch check
```

ИИ-анализ (`--ai`) показывает каждое принятое значение с файлом-источником и
дословной цитатой; конфликты с уже заполненными полями не применяются, а
выводятся на решение эколога. Провайдеры: локальные **Ollama**/**LM Studio**
и внешние **Anthropic, OpenAI, OpenRouter, DeepSeek, Gemini, Groq, Mistral,
xAI, Together, VseGPT, ProxyAPI, GigaChat, YandexGPT** (ключ — переменной
окружения, порядок запасных задаётся в `~/.ecodoc/config.json`).

## Контур «Отчётность» (формы)

| Код              | Форма                                          | Статус |
|------------------|------------------------------------------------|--------|
| `declaration-nvos` | Декларация о плате за НВОС                    | ✓ полностью (XML + Excel + расчёт платы) |
| `waste-movement`   | Учёт отходов / движение (Приказ №1028)       | ✓ XML + Excel + проверка баланса |
| `pek`              | Отчёт по ПЭК (Приказ №109)                   | ✓ XML + Excel (программа/результаты — `extra.pek`) |
| `cadastre-spb`     | Кадастр отходов (СПб/ЛО)                     | ✓ XML + Excel (срок региональный — сверять) |
| `2tp-waste`        | 2-ТП (отходы)                                | ✓ XML + Excel (в ЛКПП есть API) |
| `2tp-air`          | 2-ТП (воздух)                                | ✓ XML + Excel (в ЛКПП есть API) |
| `waste-report-iii` | Отчётность по отходам (III кат./МСП)         | ✓ XML + Excel (получатели — `extra.waste_receivers`) |
| `2tp-water`        | 2-ТП (водхоз)                                | ✓ XML + Excel (водозабор/водоотведение/оборот — форма «Данные») |

| Команда | Документ | Статус |
|---|---|---|
| `passport` | Паспорта отходов I–IV класса (.docx из ФККО) | ✓ авто |
| `volume ndv\|nds\|szz` | Тома НДВ/НДС/СЗЗ (.docx, импорт из «Эколога») | ○ каркас сборки — см. [docs](docs/Разработка_томов_НДВ_НДС_СЗЗ.md) |
| `dispersion` | Профиль рассеивания по оси (МРР-2017, точечный источник) | ✓ эксп. |
| `dispersion-map` | Карта рассеивания: суммация источников + изолиния ПДК (SVG) | ✓ эксп. — см. [УПРЗА_анализ](docs/УПРЗА_анализ.md) |
| `upraza-export` | Выгрузка источников для загрузки в УПРЗА («Эколог») — Excel+JSON | ✓ |
| — | ПНООЛР, класс опасности, НМУ, ПЛАРН, ДВОС, инвентаризация выбросов, программа ПЭК, ТУ | ○ каркасы (видны в `list`) |

## Модуль «Календарь подачи»

`calendar` — по категории объекта НВОС и видам воздействия выдаёт график сроков
на год и чек-лист обязательных к наличию документов.

```bash
python -m ecodoc calendar -i context.json --year 2026 --xlsx out/calendar.xlsx
python -m ecodoc passport -i context.json -o out/passports
python -m ecodoc volume ndv -i context.json --sources-xlsx ист.xlsx --appendix КХА.pdf
```

## Установка

**Автоустановщик (рекомендуется):** двойной щелчок по `install.bat` —
проверит Python, создаст `.venv`, поставит зависимости и команду `ecodoc`,
найдёт локальные ИИ (Ollama — все модели, LM Studio) и ключи внешних API,
подскажет, чего не хватает (Tesseract, Ollama).

Вручную:

```bash
cd C:\Users\veter\ЭкоДок
pip install -e ".[ocr,pdf]"
ecodoc ai setup
```

`PyMuPDF`, `openpyxl`, `lxml`, `python-docx` — обязательны.
`pytesseract` + установленный **Tesseract-OCR** (с пакетом `rus`) — только для
распознавания сканов/jpg.

## Рабочий сценарий (гибрид: парсинг + ручная доводка)

```bash
# 1. Распознать реквизиты из приложенных документов в context.json
python -m ecodoc analyze "договор.pdf" "акты.docx" "скан_лимитов.jpg" -o context.json

# 2. Открыть context.json, проверить распознанное и дозаполнить массы/объёмы

# 3. Сгенерировать XML (для ЛКПП) и .xlsx (печатная форма)
python -m ecodoc generate declaration-nvos -i context.json -o out
```

Прочие команды:

```bash
python -m ecodoc list                                  # список форм
python -m ecodoc validate declaration-nvos -i context.json
```

Готовый пример данных — `samples/example_context.json`:

```bash
python -m ecodoc generate declaration-nvos -i samples/example_context.json -o out
```

## Архитектура

```
ecodoc/
  core/        models (единый ReportContext), money (Decimal), registry,
               refdata (загрузка справочников), serialize (JSON ↔ контекст)
  parsers/     text_extract (pdf/docx/doc/jpg + OCR), extractor (реквизиты → контекст)
  render/      xlsx, xmlutil (генерация + валидация по XSD)
  reports/     base + по одному пакету на форму (declaration_nvos, waste_movement, …)
data/          rates_nvos.json, coefficients.json — ставки и коэффициенты (правит эколог)
samples/  tests/  out/
```

Новая форма добавляется одним пакетом в `reports/` с классом `@register` —
остальной код менять не нужно.

## ⚠ Что проверить перед сдачей

1. **Ставки и коэффициент индексации** в `data/rates_nvos.json`
   (`indexation`) — значения приведены по ПП РФ №913 (база 2018 г.) для примера.
   Сверьте с действующим Постановлением Правительства на отчётный год.
2. **Перечень веществ** в `rates_nvos.json` — добавьте недостающие коды.
3. **XSD-схемы РПН** не входят в комплект. Положите официальную `.xsd` и
   проверьте XML: `ecodoc.render.xmlutil.validate_against_xsd(...)`.
   Текущий XML — самоописательная структура, требует маппинга на актуальную XSD.

## Формула платы за НВОС

```
плата = масса × ставка × Кинд × Кполоса × Кдоп
```

- *Кполоса*: в пределах норматива = 1, лимита = 25, сверх = 100 (выбросы/сбросы);
  отходы — в пределах лимита = 1, сверх = 5;
- *Кинд* — индексация ставок;
- *Кдоп* — территория (ООПТ ×2) и спец-коэффициенты за отходы.

Основание: ст. 16.3 ФЗ-7, ПП РФ №913, ПП РФ №255.
