# История версий

## 0.8.0 — 2026-07-05

Удобство интерфейса по итогам dex-ревью UX (DeepSeek+Mistral — оба
единогласно: сырой JSON во «Данных» — блокер для эколога-непрограммиста):

- **Вкладка «Данные» — форма вместо сырого JSON**: разделы «Реквизиты
  организации», «Отчётный период», редактируемые таблицы «Объекты НВОС»,
  «Выбросы и сбросы», «Отходы» (добавить/удалить строку). Правка полей
  без риска сломать структуру. Сырой JSON остался в сворачиваемом блоке
  «Расширенно» для опытных.
- **Вкладка «Рассеивание» — таблица источников** с понятными
  подписями-подсказками (H — высота трубы, w₀ — скорость ГВС и т.д.)
  вместо ручного JSON; коэффициент A подставляется сам.
- **Подсказка «что дальше»** после приёма документов: ссылки на «Данные»
  и «Генерация».
- Крупнее базовый шрифт (15px), насыщеннее акцентный и красный цвета —
  читаемость для возрастных пользователей.

## 0.7.2 — 2026-07-05

Прогон тестов приложения (17/17) + dex-ревью (DeepSeek+Mistral,
находки сверены с кодом; отклонены ложные — «двойное округление»
итогов на деле идемпотентно, у отходов корректно 2 корзины):

- **Исправлен data-flow организации**: правки реквизитов (ИНН, ОКТМО,
  адрес…) во вкладке «Данные» теперь сохраняются в org.json и
  переживают перезагрузку — раньше молча терялись при генерации.
- **Проверка контрольной суммы ИНН и ОГРН**: опечатка/ошибка OCR в ИНН
  плательщика ловится при валидации декларации (ошибка), неверный
  ОГРН — предупреждение.
- **Предупреждение о сроке сдачи**: при генерации формы — просрочка
  (со ссылкой на ст. 8.5 КоАП) или напоминание, если срок ≤ 30 дней.
- Мягкие санити-проверки в расчёте платы: коэффициент территории < 1,
  рассинхрон года ставок (если год указан в справочнике).
- Тесты: контрольные суммы реквизитов и подсказки о сроке (17 passed).

## 0.7.1 — 2026-07-05

- **Распаковка архивов при приёме**: zip — нативно (с починкой русских
  имён из cp866), rar/7z — через установленный 7-Zip; вложенные архивы
  до 2 уровней. Документы из архивов принимаются с префиксом имени
  архива для прослеживаемости.
- Защита распаковки (по итогам dex-ревью DeepSeek+Mistral, находки
  сверены с кодом): бюджет распаковки 2 ГБ и 5000 файлов (анти-zip-бомба),
  срез пути до имени файла (анти-zip-slip) + проверка containment,
  уникальные имена при совпадении basename (не затирают друг друга),
  пропуск симлинков, устойчивость к битым/зашифрованным записям,
  сбор частично распакованного при ошибке 7-Zip.
- Тесты приёма архивов (12 passed).

## 0.7.0 — 2026-07-05

По итогам теста на реальной папке (618 файлов):

- **Пакетная загрузка больших папок**: файлы уходят партиями ~20 МБ
  с прогрессом «партия i/N · принято X/Y» — исправлена ошибка
  «Invalid string length» при загрузке сотен файлов разом.
- **Площадка = полный адрес** (вместо автоматической «Основной»):
  при создании организации указывается адрес площадки (или берётся
  юрадрес из ЕГРЮЛ); рядом с выбранной площадкой — кнопка
  «🗺 на карте» (Яндекс.Карты).
- **Генерация разделена** на контуры «Отчётность» (формы для сдачи)
  и «Разработка» (проекты и документы).
- **Отчёт приёма — раскрывающиеся секции**: «Полнота данных по формам»
  и «Распознано» открыты сразу, длинное «Распределение файлов»
  свёрнуто; после загрузки видно, что принято и чего не хватает.
- Предупреждение при ИИ-анализе больших пакетов (>25 файлов) с выбором
  «с ИИ / без ИИ».

## 0.6.0 — 2026-07-04

Свой расчёт рассеивания вместо УПРЗА + выгрузка для УПРЗА:

- **Карта рассеивания** (`dispersion-map`, вкладка «Рассеивание» в GUI):
  суммация вкладов нескольких источников с координатами, перебор
  направлений и скоростей ветра (МРР-2017: r(u/uм), p(u/uм), поперечное
  s2), поле максимальных приземных концентраций, учёт фона, доля ПДК.
  Векторная карта (SVG) с изолинией превышения ПДК и маркерами
  источников — без внешних зависимостей.
- **Выгрузка для УПРЗА** (`upraza-export`, кнопка в GUI): Excel с листами
  «Источники»/«Выбросы»/«Инструкция» под мастер импорта «Эколога» +
  машиночитаемый JSON.
- **Проверка контрагента по ИНН** (вкладка «Сервис»): статус юрлица из
  ЕГРЮЛ + ссылка на реестр лицензий ГИС ТОР КНД (открытого API лицензий
  на отходы нет — проверка вручную по ссылке).
- **ОКТМО по адресу** через DaData (нужен бесплатный токен DADATA_TOKEN):
  автозаполнение при создании организации по ИНН; частый недостающий
  реквизит платы за НВОС.

Roadmap (осталось): линейные/площадные источники и застройка в расчёте
рассеивания; 2-ТП водхоз; API ЛКПП; кнопки принятия/отклонения
конфликтов ИИ.

## 0.5.2 — 2026-07-04

Итог многомодельного ревью (dex-дебат: DeepSeek + Mistral + Claude,
3 раунда критики, все находки сверены с кодом):

- Кэш ЕГРЮЛ на 30 дней (~/.ecodoc/egrul_cache.json) — повторный ввод
  того же ИНН мгновенный и не зависит от доступности ФНС.
- Приём: временные файлы загрузки удаляются сразу после обработки;
  отчёт каждого приёма сохраняется в attachments/приём_<дата>.txt
  (история, что и когда приносили).
- Классификация: платёжные поручения больше не распознаются
  как «договор» (порядок правил).
- Календарь: экспорт в .ics (Outlook/Google/Яндекс) с напоминаниями
  за 7 дней — кнопка в GUI и engine.export_ics().
- GUI: кнопка «Проверить всё» — валидация контекста сразу под все
  7 реализованных форм; таймауты запросов (зависший сервер больше
  не крутит спиннер вечно); лимит тела запроса 512 МБ;
  понятная ошибка при неизвестной форме.
- workspace.slug() — публичный API вместо приватного _slug.

## 0.5.1 — 2026-07-03

- **Организация по одному ИНН**: реквизиты (название, КПП, ОГРН, адрес,
  руководитель) подтягиваются из открытого сервиса ФНС (ЕГРЮЛ/ЕГРИП) —
  в GUI кнопка «Создать по ИНН», в CLI `ecodoc org add --inn 780...`.
- При создании организации автоматически создаётся площадка «Основная»
  (раньше организация без площадки «зависала» — нечего было выбрать).
- GUI: клик по организации выбирает её (и единственную площадку сразу);
  у организаций без площадок — подпись «нет площадок».
- **Загрузка папки целиком** («Папку целиком…»): берутся все поддерживаемые
  файлы (pdf/doc/docx/xls/xlsx/jpg/png/tif/txt/xml/rtf), лишнее
  пропускается с пометкой.
- **Распределение файлов при приёме**: для каждого файла — что это за
  документ (акт утилизации, протокол КХА, паспорт отхода, выписка ЕГРЮЛ,
  лицензия, журнал учёта и т.д.), какие данные из него можно взять и
  каким формам он пригодится; в конце — чего не хватает.

## 0.5.0 — 2026-07-03

- **Графический интерфейс**: команда `ecodoc gui` поднимает локальный
  веб-интерфейс (127.0.0.1:8737) и открывает браузер. Без новых
  зависимостей (stdlib http.server). Вкладки: Приём документов
  (загрузка файлов + ИИ-анализ), Данные (редактор context.json),
  Генерация (все формы, проверка/генерация/PDF в один клик),
  Календарь, Рассеивание (МРР-2017), Сервис (настройка ИИ, сторож форм).
  Организации и площадки — в боковой панели.
- Защита GUI-сервера: только 127.0.0.1, проверка Host и
  Content-Type (анти-CSRF/DNS-rebinding), открытие папок — только
  внутри рабочего пространства.
- Рабочее пространство по умолчанию — `%USERPROFILE%\ЭКО.DOC`
  (одно место данных при запуске из любой папки; $ECODOC_WORKSPACE
  и существующий ./ecodoc_workspace уважаются).

## 0.4.0 — 2026-07-02

- **ИИ-слой** (`ecodoc ai`): автообнаружение локальных моделей (Ollama —
  используются все найденные chat-модели по цепочке fallback, LM Studio)
  и 13 внешних провайдеров (Anthropic, OpenAI, OpenRouter, DeepSeek, Gemini,
  Groq, Mistral, xAI, Together, VseGPT, ProxyAPI, GigaChat, YandexGPT);
  ключи — только через переменные окружения.
- **ИИ-анализ входящих** (`intake --ai`): извлечение реквизитов, актов
  утилизации, протоколов КХА/биотеста с дословной цитатой-источником для
  каждого значения; конфликты не перезаписываются, а показываются.
- **Мульти-организации × площадки** (`org add/list`, `site add`,
  `--org/--site` во всех командах): рабочее пространство на диске,
  реквизиты организации наследуются всеми её площадками.
- **Приём документов** (`intake`): реестр принятых файлов (sha1, дата),
  отчёт «что распознано / чего не хватает / какие документы ещё нужны»
  по каждой форме.
- **Сторож форм** (`watch check`): слежение за страницами приказов, ставок
  НВОС, ЛКПП, региональных кадастров (СПб/ЛО) и локальной папкой XSD.
- **Свой расчёт рассеивания** (`dispersion`) по МРР-2017: точечные
  источники, cм/xм/uм, профиль по оси, доля ПДК — см. docs/УПРЗА_анализ.md.
- **PDF-экспорт** (`pdf`, `generate --pdf`): MS Office COM → LibreOffice →
  текстовый снимок PyMuPDF.
- Каркасы контура «Разработка»: ПНООЛР, расчёт класса опасности, НМУ,
  ПЛАРН, ДВОС, инвентаризация выбросов, программа ПЭК, ТУ.
- Индекс эталонных документов (`data/etalon_index.json`) для верификации
  генераторов и обучения ИИ-анализа — локальный файл, в репозиторий
  не входит.
- Автоустановщик `install.bat`/`install.ps1`, пакетирование `pyproject.toml`
  (команда `ecodoc`), UTF-8 вывод в консоль Windows.
